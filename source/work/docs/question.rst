1.  思考一下，一天的工作时间安排上，最消耗你时间的事情是哪些？包括但不限于下面的例子。 候选举例如下

    - 问题追查，线上问题处理；
    - 新功能开发；
    - 老系统迭代优化；
    - 其他非工作事宜；

    另外，针对你的时间成本，觉得有什么办法来优化？::

        数据架构方面：
        - 讨论是的成本比较高。
        拉取模块：
        - 线上问题处理。
        - 新功能的开发。

    ::

        方法：
        - 找人分担我的任务。

#.  思考一下，现在完成的工作，相比三个月或是半年前，更多了？还是更少了？ ::

        更多了，但我希望更少而精。
        例如搜素相关性提升、语言探测准确性等任务，它们很少，但投入的精力需要更多，对个人成长更有帮助。

#.  组内的事情，主要有：大规模爬虫系统、数据平台建设、业务系统（数据分析引擎+前端），AI，对现在的工作安排是否满意？你最想从事什么事情？对于想做的事情，是否做了足够多的功课？::

        满意，暂时没有想将注意力投放于别的方向。

#.  你有优先级的概念吗？你是否为日常工作建立了一套固有程序？而不是基本乱序处理？::

        建立了，并不断完善中。
        -   线上的Case
        -   出现的Bug。
        -   新功能
        -   主要模块主要任务：相关性优化、语言探测准确度。
        -   模块优化：索引结构优化、EsBehvoir优化、拉取模块优化。

#.  和入职之前的个人相比，你觉得最近技术上时候有提升，提升是否如预期？如果有，是什么事情让你技术能力有所提升？ 如果没有，有什么计划让自己提升？团队可以做什么？公司可以做什么？进一步，如果你要离职，出去找工作，你觉得自己可以相对轻松的拿下BATJ的offer吗？::

        -   没有，所以经常感到恐慌。
        -   减少任务量，集中解决高质量的问题。有些时间关注一些刊物和技术。整理自己的博客。
        -   关于BATJ的问题，还是不要回答了，我的心里会好受些。

#.  你对相关的业务，是否有所理解吗？从提供数据产品的层面讲，你是否比你的客户更显得专业？::

        关于这方面，我不会主动投入更多的精力。不过对于搜索功能，我会加大使用次数，在搜索业务上投入更多精力。

#.  问题

    #.  提问者：我的问题语言组织是否到位？我是只想要一个简单直接的答案，还是说也想通过了解思考过程来学会思考方法？
    #.  回答者：如果我有现成答案，该不该直接给出？如果对方的问题问得不好，我该不该先引导对方把问题理顺？
    #.  时常问自己这个问题：如果过程错误，但结果正确，那这算正确还是错误？
