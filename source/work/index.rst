工作过程中的一些感悟

- 最重要的事：明确自己每个季度（3个月）最重要的一件事情，他们是你每年年终总结最重要的支柱。保证普通工作的完成只能让你不会丢掉岗位，漂亮的完成每季度“最重要的事”才会让你更近一步。
- 自己对某项工作有了想法之后，不要将“意见”提交上去后就等待“被”告知去执行。要主动的去推动它，自己活动范围内去做出成果。第一个看到螃蟹的人很多，第一个吃螃蟹的人才有价值。如：

  - Code Style
  - 不要让文档与代码分离。
  - 重视并规范Log。

- “推动”工作过程中，要积极、主动、争取。关注每个人的进度，不强制立即执行但也不要害怕招人厌烦（所有人都知道，这是工作）。
- 工作过程中，不但要对代码细心也要对数据指标细心。监控、关注各个关键数据的变动，以便及时发现项目的运行情况。很多情况，可能会因为没有预料到所以Log没有捕获，但是数据中一天会体现出来。
- 工作中，分配好工作优先级很重要。

  - 关注与上下游的交流，看清优先级。
  - 对任务做好减法，保证“代办列表”的长度。
  - 避免任务间的频繁切换

- 学会控制好自己的情绪很重要，及时察觉对方的情绪也很重要。
- 管理工作中，关注任务指标和时间点就能对同事的工作起到督促作用。
- 模块的对外接口，整理出文档比口头交接更有意义。
